# MPBP440 V12.1 — Analytics privées et commentaires

## Architecture

Le back-office existant `admin-440-mpbp-corp/` est conservé. Son ancienne vérification locale a été remplacée par Supabase Auth : une session n’est ouverte que si le jeton est valide et que l’utilisateur possède une ligne active `admin` dans `public.admin_users`.

Les statistiques privées ne sont accessibles que par des RPC `security definer` qui appellent `require_active_admin()`. Aucun tableau brut de visites, vues, likes, sessions ou commentaires n’est lisible par `anon` ou `authenticated`.

## Migration

`supabase/migrations/20260720000200_admin_analytics_comments.sql` ajoute :

- `admin_users` : rôle unique `admin`, activable/désactivable ;
- `comments` : contenu limité aux quatre clips publics, statut `pending` par défaut, historique de modération et index ciblés ;
- RLS sur les deux tables, sans politique de lecture directe ;
- RPC publiques : `submit_comment`, `get_approved_comments` ;
- RPC administratives : synthèse, clips, artistes, pages, activité, liste et modération des commentaires.

Les limites de commentaires sont de 30 secondes entre deux envois et cinq envois par visiteur et par heure. L’interface inclut aussi un champ anti-robot invisible. Les visiteurs ne peuvent lire que les commentaires `approved`.

## Interface

- `admin-440-mpbp-corp/` : onglets **Analytics** et **Commentaires**, accessibilité clavier native et textes d’état `aria-live` ;
- `mpbp-tv/index.html` : liste des commentaires approuvés et formulaire d’envoi pré-modéré ;
- aucun changement aux clips, MP4, ancres publiques ou navigation existante.

## Mise en service sécurisée

La migration doit être appliquée avant de pouvoir attribuer le premier rôle. Après application, créer le premier utilisateur dans **Supabase Auth**, puis attribuer son UUID via le SQL Editor Supabase :

```sql
insert into public.admin_users (user_id, role, is_active)
values ('UUID_DE_L_UTILISATEUR_AUTH', 'admin', true)
on conflict (user_id) do update set role = 'admin', is_active = true;
```

Ne placer ni mot de passe, ni clé d’administration, ni jeton dans le dépôt. La valeur qui était historiquement codée dans l’ancienne interface doit être considérée comme compromise et ne doit pas être réutilisée.
