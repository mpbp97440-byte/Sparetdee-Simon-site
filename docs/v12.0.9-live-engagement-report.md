# MPBP440 V12.0.9 — vues, J’aime et visites Supabase

## Architecture active

Le client statique `assets/js/mpbp440-supabase.js` appelle directement les RPC Supabase via l’URL publique fournie et une Publishable Key. Aucune clé administrative, mot de passe ou token GitHub n’est livré au navigateur.

La migration V12.0.9 complète V12.0.8 sans modifier la migration déjà appliquée :

- `sessions` reçoit un `visitor_id` anonyme ;
- `site_visits` devient unique par session et chemin normalisé ;
- `video_likes` devient unique par clip et visiteur ;
- les RPC existantes reçoivent session + visiteur, et `get_video_engagement()` retourne aussi l’état `liked` du visiteur.

## Comptage

- Un `visitor_id` UUID persiste dans `localStorage`; un `session_id` UUID vit dans `sessionStorage`.
- Aucun nom, e-mail, IP applicative, géolocalisation ou empreinte d’appareil n’est enregistré.
- Une visite est dédupliquée par session et chemin, sans hash ni paramètres de cache.
- Une vue MPBP TV est envoyée après 10 secondes de lecture cumulée, ou 20 % pour un clip de moins de 50 secondes, ou à la fin. Une vue est unique par clip et session.
- Un J’aime est atomique, réversible et unique par clip + visiteur. Les totaux viennent toujours du serveur.
- Supprimer les données du navigateur ou changer d’appareil crée volontairement un nouveau visiteur anonyme : c’est une limite normale d’un site sans compte.

## Pages injectées

- Accueil (y compris Sorties et À venir), MPBP TV, Actualités, Galerie, Événements et Music Hub : visites discrètes.
- Pages Sparetdee Simon, Makéda Muse et Juste Une Plume : visite + consultation artiste.
- MPBP TV : lecture des vues/J’aime de chaque clip, bouton J’aime accessible, compteurs sur le clip actif et les cartes.

## Sécurité et mode dégradé

RLS reste active. Les écritures anonymes directes sont interdites ; seules les RPC `SECURITY DEFINER` avec paramètres validés sont exécutables. Les anciens RPC anonymes sans `visitor_id` sont révoqués.

Le client applique un délai réseau de 5,5 secondes et protège les requêtes d’écriture concurrentes. Si le service est indisponible, le site, ses ancres et ses vidéos fonctionnent normalement ; le bouton est désactivé et le message « Statistiques temporairement indisponibles » est affiché. La preview lit les compteurs mais n’enregistre jamais visites, vues ou J’aime.

## Tests

- Validation syntaxique des JavaScript modifiés avec `node --check`.
- Validation des espaces avec `git diff --check`.
- Lecture distante initiale V12.0.8 : `l-argent` = 0 vue, 0 J’aime avant intégration V12.0.9.
- Les tests de mutation distante sont exécutés seulement après la migration de production, avec un visiteur contrôlé puis retrait du J’aime ; aucune vue artificielle ne sera injectée dans les compteurs publics.
- La migration est validée dans une pile Supabase isolée par GitHub Actions avant toute migration distante.

## Limites

Sans authentification, les UUID de navigateur apportent une protection anti-doublon raisonnable, pas une prévention absolue contre un acteur malveillant qui efface son stockage local ou fabrique des UUID.
